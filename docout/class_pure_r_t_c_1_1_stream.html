Detailed Description

Handles the WebRTC (audio, video) stream, identifies the stream, and identifies the location where the stream should be displayed.

There are two stream classes: LocalStream and RemoteStream.

void PureRTC.Stream.close ( )
inline

This function closes the stream.


Remarks:
If the stream has audio and/or video, it also stops capturing camera/microphone. If the stream is published to a conference, the function also un-publishes it. If the stream is published to a P2P session, the function does NOT un-publish it. If the stream is showing in an HTML element, the stream would be hidden. Once a LocalStream is closed, it is no longer usable.



<script type="text/JavaScript">
var stream = PureRTC.Stream({audio:true, video:true, data: false, attributes:
{name:'PureRTCStream'}});
stream.close();
</script>

boolean PureRTC.Stream.disableAudio ( )
inline

This function disables underlying audio track in the stream if it has audio capacity; otherwise it does nothing.


Remarks:
For remote stream, it stops decoding audio; for local stream, it also stops capturing audio. But capturing cannot be stopped currently in Edge by this API.

Returns
true The stream has audio and the audio track is enabled previously;
otherwise false.
